# Python Exception Handling

This folder has 25 Python files covering different types of exceptions and how to handle them. Each file focuses on one specific error type with simple examples.

## The Files

**01_value_error.py** - Converting invalid strings to numbers (like "hello" to int)

**02_type_error.py** - Adding incompatible types together (string + number)

**03_zero_division_error.py** - The classic divide by zero problem

**04_file_not_found_error.py** - Trying to open files that don't exist

**05_index_error.py** - Accessing list indices that are out of range

**06_key_error.py** - Looking for dictionary keys that aren't there

**07_attribute_error.py** - Calling methods or attributes that don't exist

**08_import_error.py** - Importing modules that Python can't find

**09_name_error.py** - Using variables before defining them

**10_keyboard_interrupt.py** - Handling Ctrl+C interruptions

**11_overflow_error.py** - Numbers getting too large to handle

**12_recursion_error.py** - Functions calling themselves too many times

**13_assertion_error.py** - Failed assert statements

**14_stopiteration_error.py** - Iterator running out of items

**15_general_exception.py** - Catching any exception type

**16_multiple_exceptions.py** - Catching multiple exception types at once

**17_multiple_except_blocks.py** - Using multiple except blocks for different errors

**18_try_except_else.py** - Using the else block (runs when no error occurs)

**19_try_except_finally.py** - Using the finally block (always runs)

**20_try_except_else_finally.py** - Complete structure with all blocks

**21_raising_exceptions.py** - Manually raising exceptions with the raise keyword

**22_custom_exceptions.py** - Creating your own exception classes

**23_nested_try_except.py** - Try-except blocks inside other try-except blocks

**24_runtime_error.py** - Generic runtime errors

**25_ioerror.py** - File and I/O operation errors

## Running the Files

Just run any file with `python filename.py` - they all work independently. Start with the numbered ones from 01 onwards if you want to follow a logical progression.
